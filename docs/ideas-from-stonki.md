### 2. Память о пользователе (User Memory)

**Критически важная концепция!** Советник должен помнить:

```yaml
# data/user-memory.yaml
profile:
  risk_tolerance: moderate
  horizon: medium
  experience: intermediate
  focus: dividend_growth

history:
  recipes:
    - id: recipe-001
      ticker: SBER
      created: 2026-01-15
      closed: 2026-02-20
      result: +8.5%
      thesis: "Дивидендная игра"
      
    - id: recipe-002
      ticker: LKOH
      created: 2026-02-01
      status: active
      
  lessons_learned:
    - date: 2026-01-10
      event: "Продал SBER слишком рано, упустил +5%"
      lesson: "Ждать целевого уровня, не продавать на эмоциях"
      
    - date: 2026-02-15
      event: "Не поставил стоп на GAZP, потерял -8%"
      lesson: "Всегда ставить стоп-лосс"

preferences:
  avoid_tickers: [MGNT, POLY]  # неудачный опыт
  favorite_sectors: [financial, energy]
  max_position_size: 15  # %
```

**Использование памяти:**
```
Пользователь: "Что думаешь про GAZP?"

Советник проверяет память:
→ Пользователь уже торговал GAZP?
→ Какой был результат?
→ Какие уроки извлечены?
→ Соответствует ли профиль риска?

Ответ: "GAZP сейчас выглядит привлекательно (P/E 3.2), но 
учитывая твой прошлый опыт с этой бумагой (потеря -8% без стопа),
рекомендую либо пропустить, либо войти с жёстким стопом."
```

### 3. Scheduled Prompts (Периодический анализ)

**Мощная концепция:** не просто алерт, а периодический полный анализ:

```yaml
# config/monitoring.yaml
tasks:
  - name: "Еженедельный анализ портфеля"
    type: recurring_prompt
    schedule: "FREQ=WEEKLY;BYDAY=FR;BYHOUR=18"
    prompt: |
      Проанализируй мой портфель:
      
      1. Проверь отклонения от целевого распределения
      2. По каждой позиции проверь:
         - Тренд (бычий/медвежий/боковик)
         - RSI (перекупленность/перепроданность)
         - Ближайшие уровни поддержки/сопротивления
      3. Проверь календарь событий на следующую неделю
      4. Есть ли рекомендации по ребалансировке?
      
      Учитывай мои прошлые решения и уроки из памяти.

  - name: "Ежедневная проверка сигналов"
    type: recurring_prompt
    schedule: "FREQ=DAILY;BYHOUR=10;BYMINUTE=0"
    prompt: |
      Проверь торговые сигналы по моим активным рецептам:
      - Достигнут ли уровень входа?
      - Достигнут ли стоп-лосс?
      - Достигнута ли цель?
      - Есть ли изменения в технической картине?

  - name: "Алерт SBER 275"
    type: price_cross
    ticker: SBER
    price: 275.00
    direction: above
    prompt: |
      Цена SBER пересекла 275. Это уровень сопротивления.
      Проанализируй:
      1. Есть ли пробой или отскок?
      2. Стоит ли фиксировать часть позиции?
      3. Обнови рецепт если нужно.
```

### 8. Уроки из сделок (Lessons Learned)

После закрытия рецепта — анализ:

```bash
./bin/skill recipe:close --id=xxx --result=profit --actual_pnl=+8.5%
```

Советник спрашивает:
```
Что пошло хорошо?
□ Технический анализ был точным
□ Вошёл по хорошей цене
□ Дождался цели

Что можно улучшить?
□ Рано продал
□ Слишком большой размер позиции
□ Не учёл новость

Какой урок на будущее?
[свободный ввод]
```

Урок сохраняется в память и учитывается в будущих рекомендациях.

## Главный инсайт

**Stonki AI = Рецепты + Память + Мониторинг**

Не просто "проанализируй и дай рекомендацию", а:
1. Создай рецепт с планом
2. Следи за исполнением (мониторинг)
3. Запомни результат (память)
4. Учитывай в будущих решениях

Это превращает разовый совет в **развивающуюся систему**.
